# 11. Maximum Product Subarray

Given an integer array `nums`, find a subarray that has the largest product, and return *the product*.

The test cases are generated so that the answer will fit in a **32-bit** integer.

**Note** that the product of an array with a single element is the value of that element.

**Example 1:**

```
Input: nums = [2,3,-2,4]
Output: 6
Explanation: [2,3] has the largest product 6.
```

**Example 2:**

```
Input: nums = [-2,0,-1]
Output: 0
Explanation: The result cannot be 2, because [-2,-1] is not a subarray.
```

sol:

So the brute force way would be go on each number make a sub array from there and save it’s product. And then which ever is the highest product return that as output. That would just take so much time.

We can solve this with dynamic programming.

For each index, we track:

- the maximum product of a subarray ending at this index
- the minimum product of a subarray ending at this index

We need both because a negative number can flip signs. A very small negative product, when multiplied by another negative number, can become a very large positive product.

So for each number:

- if the current number is negative, we swap max and min first
- then we update:
    - max_here = max(current, current * max_here)
    - min_here = min(current, current * min_here)

We keep a global answer storing the maximum product seen so far.

The reason swap works is:

- multiplying by a negative reverses ordering
- the old maximum can become the new minimum
- the old minimum can become the new maximum

So the final answer is the largest `max_here` seen during the traversal.
